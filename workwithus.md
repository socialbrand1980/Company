# Product Requirements Document (PRD)

# Work With Us Page

SocialBrand 1980 Digital Agency

---

# 1. Overview

Halaman **Work With Us** adalah halaman konversi utama yang digunakan untuk mengubah visitor website menjadi **qualified client leads**.

Halaman ini memungkinkan calon klien untuk:

* Memilih layanan yang mereka butuhkan
* Memahami detail layanan agency
* Mengisi informasi brand mereka
* Mengirim brief awal project

Halaman ini berfungsi sebagai **client qualification system** sebelum proses **Brand Discovery Call**.

---

# 2. Objectives

Tujuan halaman ini adalah:

1. Mengumpulkan **brand brief awal** dari calon klien
2. Menjelaskan layanan agency secara lebih detail
3. Menyaring klien berdasarkan **kebutuhan dan budget**
4. Mengumpulkan **qualified leads**
5. Mempersiapkan informasi sebelum discovery call

---

# 3. User Flow

User Flow halaman:

User membuka website
↓
User klik tombol **Start a Project / Work With Us**
↓
User memilih **service** yang diinginkan
↓
Detail service muncul secara dinamis
↓
User membaca detail layanan
↓
User mengisi **Brand Information Form**
↓
User submit form
↓
User diarahkan ke **Confirmation Page**

# 15. Lead Management System

Semua data yang dikirim melalui form **Work With Us** akan disimpan sebagai **lead database**.

Sistem penyimpanan leads menggunakan **Google Spreadsheet** sebagai database sederhana untuk mengelola calon klien.

Tujuan penggunaan spreadsheet:

* Menyimpan seluruh data leads
* Memudahkan proses follow-up
* Menjadi CRM sederhana untuk agency
* Mengorganisir pipeline project

---

# 16. Google Spreadsheet Integration

Setiap form submission dari halaman **Work With Us** akan otomatis mengirim data ke **Google Spreadsheet**.

Flow sistem:

User submit form
↓
Next.js API menerima data
↓
API mengirim data ke Google Sheets API
↓
Data ditambahkan sebagai **row baru** di spreadsheet

---

# 20. Required Environment Variables

Aplikasi membutuhkan environment variables berikut:

ID dari Google Spreadsheet.

Credential JSON dari Google Service Account.

# 21. Google Cloud Setup

Langkah konfigurasi:

1. Membuat project di Google Cloud Console
2. Mengaktifkan Google Sheets API
3. Membuat Service Account
4. Mengunduh credential JSON
5. Menyimpan credential sebagai environment variable
6. Memberikan akses Editor pada spreadsheet kepada service account
